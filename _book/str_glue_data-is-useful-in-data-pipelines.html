mtcars %>% str_glue_data(“{rownames(.)} has {hp} hp”)

str_flatten(c(“x”, “y”, “z”)) str_flatten(c(“x”, “y”, “z”), collapse = “,”) str_flatten(c(“x”, “y”, “z”), collapse = “,”, last = “, and”)

7.0.1 패턴으로 작업

  • 정규표현식! 문자열의 패턴을 기술하는 매우 정갈한 언어
  • str_detect() - “이 패턴이 내 벡터 어딘가에 있는가?”
  • filter()와 짝을 잘 이룸 x <- c(“apple”, “banana”, “pear”) str_detect(x, “e”) str_detect(x, “b”) str_detect(x, “x”)

babynames %>% filter(str_detect(name, “x”))

babynames %>% group_by(year) %>% summarise(prop_x = mean(str_detect(name, “x”)))

7.0.1.1 정규 표현식(regular expression; regex, regexp)

  • 주어진 문자열에 특정한 패턴이 있는 경우, 해당 패턴을 일반화(수식화)한 문자열
  • 특정 패턴을 표현한 문자열을 메타 문자(meta character)라 함
  • 일반적으로 특정 패턴을 가지는 문자열을 찾고, 다른 값으로 대체하기 위해 사용
  • 기본 메타 문자
기호? 설명
. 무엇이든 한 글자를 의미
+ + 앞에 오는 표현이 하나 이상 포함
* * 앞에 오는 표현이 0 또는 하나 이상 포함
? ? 앞에 오는 표현이 0 또는 하나 포함
^ ^ 뒤에 오는 표현으로 시작
$ $ 앞에 오는 표현으로 끝나는 경우
{}
()

  • 정규표현식 기초

  • 가장 간단한 패턴 exact - 패턴에서 문자형 서열을 그대로 포함하는 문자열을 매칭

  • ., +, *, [, ], ?과 같은 구두점(punctuation)은 특별한 의미가 있음

  • .은 임의의 문자에 매칭, 예: “a.”는 a가 있고 임의의 문자가 따라오는 문자열에 매칭 str_detect(c(“a”, “ab”, “ae”, “bd”, “ea”, “eab”), “a.”)

  • quantifiers : 한 요소가 다른 패턴에 얼마나 많이 적용될 수 있는지 컨트롤

  • str_view_all() 매치된 문자들을 <>로 둘어싸고 파란색으로 보여줌 str_view_all(c(“a”, “ab”, “ae”, “bd”, “ea”, “eab”), “a.”)

  • ?는 패턴을 선택사항으로 만듬(0 혹은 1회 매치)

  • +는 패턴을 반복(적어도 한 번 매치)

  • 은 패턴을 선택사항으로 만들거나 반복(0회를 포함하여 어떤 횟수나 매칭) str_detect(c(“a”, “ab”, “ae”, “bd”, “ea”, “eab”), “a?”) str_detect(c(“a”, “ab”, “ae”, “bd”, “ea”, “eab”), “a+”) str_detect(c(“a”, “ab”, “ae”, “bd”, “ea”, “eab”), “a”)

  • character classes

  • []가 정의하고 문자 집합을 매칭

  • [abcd]는 “a”,“b”,“c”,“d” 중 하나(이상)과 매칭

  • ^로 시작하여 매치를 뒤집을 수도 있음

  • [^abcd]는 “a”,“b”,“c”,“d”를 제외한 어느 것과도 매칭, 특정 이름에 모음을 찾기 위해 사용할 수 있음

names <- c(“Hadley”, “Mine”, “Garrett”) str_view_all(names, “[aeiou]”)

  • 매치 카운트
  • str_count()는 st_detect()의 변형 - 문자열에 얼마나 많이 매칭되어 있는지 알려줌 / st_detect()는 T/F 반환

babynames %>% count(name) %>% mutate( vowels = str_count(name, “[aeiou]”), consonants = str_count(name, “[^aeiou]”) )

–> 정규표현식은 대소문자를 구분함!
